What is JavaScript?

The language of the web — running in every browser on earth

🟣 Why

HTML builds structure. CSS styles it. JavaScript makes it alive — responding to clicks, fetching data, updating content without reloading.

🔵 How

Write code inside <script> tags or a separate .js file linked with <script src>. The browser's engine runs it instantly.

🟢 Where

Every webpage, mobile app (React Native), server (Node.js), desktop app (Electron), and even IoT devices.

📄 index.html — Your First JS
<!DOCTYPE html>
<html>
<body>
  <h1>Hello World</h1>

  <script>
    // This is a comment - browser ignores it
    console.log("JavaScript is running!");
    console.log("Welcome to the guide!");
    
    // Show a popup
    alert("Hello, World! 👋");
  </script>
</body>
</html>
Console Output
JavaScript is running!
Welcome to the guide!
> [Browser shows: "Hello, World! 👋" popup]
💡
Tip: Open browser DevTools with F12 → Console tab to see console.log output in real-time. This is your best debugging tool.

Variables

Containers for storing and reusing data

🟣 Why

Store user input, calculation results, or any value you need to use more than once. Without variables, you'd repeat yourself constantly.

🔵 How

Use let for values that change, const for fixed values, avoid var (old, buggy scoping).

🟢 Where

User's name in a form, item price in a cart, score in a game, current page number — everywhere.

📄 variables.js
// const — value never changes
const siteName = "MyShop";
const taxRate = 0.18;

// let — value can change
let userName = "Hari";
let score = 0;

// Changing a let variable
score = 10;
userName = "Hari Kumar";

console.log(siteName);          // "MyShop"
console.log(taxRate);          // 0.18
console.log(userName);         // "Hari Kumar"
console.log(score);            // 10

// Template literal — embed variables in text
console.log(`Welcome to ${siteName}, ${userName}!`);

// ❌ This would throw an error:
// siteName = "OtherShop";  → TypeError: Assignment to constant variable
Console Output

Hari Kumar
10
Welcome to MyShop, Hari Kumar!

Operators

Perform calculations, comparisons, and logic

📄 operators.js
// ── ARITHMETIC ──
console.log(10 + 5);    // 15
console.log(10 - 3);    // 7
console.log(4 * 3);     // 12
console.log(10 / 4);    // 2.5
console.log(10 % 3);    // 1  (remainder)
console.log(2 ** 8);    // 256 (power)

// ── COMPARISON — always returns true/false ──
console.log(5 === 5);   // true  (strict equal)
console.log(5 !== 3);   // true  (not equal)
console.log(10 > 7);    // true
console.log(3 >= 5);    // false

// ❗ == vs === — always use ===
console.log(5 == "5");  // true  ← loose, converts types
console.log(5 === "5"); // false ← strict, checks type too

// ── LOGICAL ──
console.log(true && false);  // false (AND — both must be true)
console.log(true || false);  // true  (OR  — one must be true)
console.log(!true);          // false (NOT — flips value)

// Real-world: check login + admin access
const loggedIn = true, isAdmin = false;
console.log(loggedIn && isAdmin);  // false — not admin
Console Output
15    7    12    2.5    1    256
true    true    true    false
true    ← loose == (avoid this!)
false   ← strict === (use this)
false    true    false

Conditionals

Make decisions — run different code based on conditions

🟣 Why

Real apps must respond differently to different situations — logged in vs guest, payment success vs failure, admin vs user.

🔵 How

Use if/else if/else for conditions. Use switch for many specific cases. Use ternary ? : for simple one-liners.

🟢 Where

Login checks, form validation, showing/hiding features, routing, pricing tiers — everywhere.

📄 conditionals.js — Real-world Examples
// ── Example 1: Grade system ──
const marks = 75;

if (marks >= 90) {
  console.log("Grade: A+");
} else if (marks >= 75) {
  console.log("Grade: A");   // ← This runs
} else if (marks >= 60) {
  console.log("Grade: B");
} else {
  console.log("Grade: F");
}

// ── Example 2: Ternary operator ──
const age = 20;
const status = age >= 18 ? "Adult" : "Minor";
console.log(status);  // "Adult"

// ── Example 3: Switch — day of week ──
const day = "Monday";
switch(day) {
  case "Saturday":
  case "Sunday":
    console.log("Weekend! 🎉");
    break;
  default:
    console.log("Weekday — back to work! 💼");
}

Loops

Repeat actions without repeating code

Arrays

Ordered lists — the most used data structure in JavaScript

📄 arrays.js — map, filter, reduce, find
const products = [
  { name: "Laptop",  price: 45000, inStock: true  },
  { name: "Phone",   price: 15000, inStock: false },
  { name: "Tablet",  price: 25000, inStock: true  },
  { name: "Monitor", price: 12000, inStock: true  },
];

// map — transform every item → new array
const names = products.map(p => p.name);
console.log(names); // ["Laptop","Phone","Tablet","Monitor"]

// filter — keep only matching items
const available = products.filter(p => p.inStock);
console.log(available.length);  // 3

// find — first matching item
const laptop = products.find(p => p.name === "Laptop");
console.log(laptop.price);  // 45000

// reduce — compute a single value
const total = products.reduce((sum, p) => sum + p.price, 0);
console.log(`Total: ₹${total}`);  // ₹97000

// sort — order by price
const byPrice = [...products].sort((a, b) => a.price - b.price);
console.log(byPrice.map(p => p.name));
// ["Monitor","Phone","Tablet","Laptop"]

// some / every
console.log(products.some(p => p.price > 40000));  // true
console.log(products.every(p => p.inStock));        // false

DOM Manipulation

Read and change HTML elements with JavaScript

🟣 Why

This is how JavaScript makes webpages dynamic — update text, show/hide elements, change styles, add items to a list without reloading.

🔵 How

Use document.getElementById, querySelector to find elements, then change their textContent, innerHTML, or style.

🟢 Where

Todo lists, shopping carts, form validation, live search, dark mode toggle, dynamic menus — every interactive UI.

📄 dom.html — Real-world Todo List
<ul id="todoList"></ul>
<input id="todoInput" placeholder="Add a task...">
<button onclick="addTodo()">Add</button>

<script>
const list = document.getElementById("todoList");
const input = document.getElementById("todoInput");

function addTodo() {
  if (!input.value.trim()) return;  // empty check

  const li = document.createElement("li");
  li.textContent = input.value;
  
  // Add delete button to each item
  const btn = document.createElement("button");
  btn.textContent = "✕";
  btn.onclick = () => li.remove();
  li.appendChild(btn);

  list.appendChild(li);  // add to page
  input.value = "";      // clear input
}
</script>

    Async / Promises / await

    Handle time-taking operations without freezing the page

    🟣 Why

    Fetching data from a server takes time. Without async, your whole page freezes waiting for it. Async lets the page stay responsive.

    🔵 How

    Mark a function async, then use await before slow operations. Errors are caught with try/catch.

    🟢 Where

    API calls, database queries, file reads, image loading, timers — anything that takes time.

    📄 async.js — Promise & async/await
    // ── Simulating a delay (like a server response) ──
function fetchUserData(userId) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        resolve({ id: userId, name: "Hari", role: "admin" });
      } else {
        reject(new Error("Invalid user ID"));
      }
    }, 1000); // 1 second delay
  });
}

// ── async/await — cleanest way to use Promises ──
async function loadUser() {
  try {
    console.log("⏳ Loading user...");
    const user = await fetchUserData(1);
    console.log("✅ Got user:", user);
    console.log(`Welcome, ${user.name}!`);
  } catch (err) {
    console.log("❌ Error:", err.message);
  }
}

loadUser();
console.log("(This runs immediately, not waiting!)");
    Console Output (with timing)
    ⏳ Loading user...
    (This runs immediately, not waiting!)
    > ... 1 second passes ...
    ✅ Got user: { id: 1, name: "Hari", role: "admin" }

    Fetch API

    Get real data from the internet — the backbone of modern apps

    📄 fetch.js — Calling a Real API
    // GET request — fetch a list of posts
async function getPosts() {
  try {
    const response = await fetch(
      "https://jsonplaceholder.typicode.com/posts?_limit=3"
    );

    if (!response.ok) throw new Error("Network error");

    const posts = await response.json();  // parse JSON

    posts.forEach(post => {
      console.log(`[${post.id}] ${post.title}`);
    });
  } catch (err) {
    console.log("Error:", err.message);
  }
}

getPosts();

// POST request — send data to server
async function createPost(title, body) {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title, body, userId: 1 })
  });
  const data = await res.json();
  console.log("Created post with id:", data.id);
}

    ES6+ Modern Features

    Newer syntax that makes JavaScript cleaner and more powerful

    📄 es6.js — Destructuring, Spread, Optional Chaining & more
    // ── Destructuring ──
const [first, second, ...rest] = [10, 20, 30, 40];
console.log(first, second, rest); // 10  20  [30, 40]

// ── Spread operator ──
const a = [1, 2], b = [3, 4];
const merged = [...a, ...b];
console.log(merged);  // [1, 2, 3, 4]

// ── Optional chaining (?.) ──
const user = { address: { city: "Hyderabad" } };
console.log(user?.address?.city);   // "Hyderabad"
console.log(user?.phone?.number);  // undefined (no error!)

// ── Nullish coalescing (??) ──
const name = null ?? "Guest";
console.log(name);  // "Guest"

// ── Set — unique values only ──
const nums = [1, 2, 2, 3, 3, 3];
const unique = [...new Set(nums)];
console.log(unique);  // [1, 2, 3]

// ── Map — key-value with any key type ──
const scores = new Map();
scores.set("Hari", 95);
scores.set("Priya", 88);
console.log(scores.get("Hari"));  // 95

    Error Handling

    Gracefully handle things going wrong — every production app needs this

    📄 errors.js — try/catch/finally
    // ── Basic try/catch ──
try {
  const data = JSON.parse("invalid json {");
} catch (err) {
  console.log("Caught error:", err.message);
} finally {
  console.log("This always runs");  // cleanup code here
}

// ── Custom error ──
function divide(a, b) {
  if (b === 0) throw new Error("Cannot divide by zero!");
  return a / b;
}

try {
  console.log(divide(10, 2));   // 5 ✅
  console.log(divide(10, 0));   // ❌ throws
} catch (err) {
  console.log("Error:", err.message);
}

// ── Error types ──
// TypeError    — wrong type (null.name)
// ReferenceError — variable not defined
// SyntaxError  — bad code syntax
// RangeError   — value out of range

    OOP & Classes

    Blueprint-based programming — build complex, reusable systems

    📄 classes.js — Class, Inheritance, Getter/Setter
    // ── Base class ──
class Animal {
  constructor(name, sound) {
    this.name = name;
    this.sound = sound;
  }
  speak() {
    return `${this.name} says ${this.sound}!`;
  }
}

// ── Inheritance with extends ──
class Dog extends Animal {
  constructor(name) {
    super(name, "Woof");  // call parent constructor
    this.tricks = [];
  }
  learn(trick) {
    this.tricks.push(trick);
    return `${this.name} learned ${trick}!`;
  }
  get summary() {
    return `${this.name} knows: ${this.tricks.join(", ") || "nothing yet"}`;
  }
}

const dog = new Dog("Max");
console.log(dog.speak());        // "Max says Woof!"
console.log(dog.learn("sit"));   // "Max learned sit!"
console.log(dog.learn("shake")); // "Max learned shake!"
console.log(dog.summary);          // "Max knows: sit, shake"
    Console Output
    Max says Woof!
    Max learned sit!
    Max learned shake!
    Max knows: sit, shake

    LocalStorage & JSON

    Save data in the browser — persists even after page refresh

    🟣 Why

    Keep user preferences, cart items, or login state without a server. Data survives page refreshes and browser restarts.

    🔵 How

    Use localStorage.setItem(key, value) to save and getItem(key) to read. Objects must be JSON-stringified.

    🟢 Where

    Dark mode preference, language setting, shopping cart, remember-me login, offline-capable apps.

    📄 storage.js — Save & Load Data
    // ── Save data ──
localStorage.setItem("username", "Hari");

// ── Save an object (must stringify) ──
const cart = [
  { id: 1, name: "Laptop", qty: 1 },
  { id: 2, name: "Mouse",  qty: 2 }
];
localStorage.setItem("cart", JSON.stringify(cart));

// ── Read data ──
const user = localStorage.getItem("username");
console.log(user);  // "Hari"

// ── Read and parse object ──
const savedCart = JSON.parse(localStorage.getItem("cart"));
console.log(savedCart[0].name);  // "Laptop"

// ── Delete ──
localStorage.removeItem("username");

// ── Clear all ──
// localStorage.clear();

// ── Safe read with fallback ──
const theme = localStorage.getItem("theme") ?? "dark";
console.log(`Theme: ${theme}`);  // "Theme: dark"
